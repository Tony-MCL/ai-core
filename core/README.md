# Workflows

## Default workflow (strict)
1) Inspect: search + read relevant files
2) Plan: list files to change + rationale
3) Patch: produce unified diff
4) Apply: user approves patch
5) Verify: run configured checks
6) Learn: suggest pattern/playbook updates (optional)

## “Explore” workflow (optional)
Same as strict, but allows more speculative suggestions.
Still patch-only and verify.
