# Memory Model

## Repo memory (in the repo)
Intended to be versioned and shared with the project.

Recommended structure:
- /petey/CONTRACT.md
- /petey/DECISIONS.md
- /petey/PATTERNS.md
- /petey/TASKLOG.md

Repo memory overrides global memory.

## Global memory (on the machine)
Transparent, editable library used across projects:
- global patterns
- playbooks
- snippets
- tool configs

Global memory is not “training”; it’s curated knowledge.
